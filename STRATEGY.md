# Стратегия st4 — описание для анализа и улучшения нейросетью

> Документ для ИИ-анализа: полное описание торговой стратегии, известные проблемы,
> логи прогонов (бэктесты на истории MOEX + живой форвард-тест в T-Bank sandbox) и
> результаты. Цель — найти, что можно улучшить.
> Сгенерировано из реального кода и данных прода. Дата среза: 2026-06-25.

---

## 1. Суть стратегии

Статистический арбитраж (**mean-reversion**) спреда между фьючерсами на **обыкновенные**
и **привилегированные** акции одного эмитента (FORTS, MOEX). Гипотеза: две бумаги одного
эмитента жёстко коинтегрированы, спред между ними колеблется вокруг средней и возвращается
к ней. Торгуем отклонения спреда от средней.

**Пары:** Сбер (SBRF/SBPR), Сургут (SNGR/SNGP), Татнефть (TATN/TATP), Ростелеком (RTKM/RTKMP).

**Данные:** MOEX ISS (свечи 10m/60m, публичные, лаг 15–30 мин) для графика/бэктеста;
T-Bank Invest API sandbox (real-time 1m/5m/10m/60m) для исполнения и детального графика.

## 2. Торговая механика

### Спред и позиция
- **Спред = Close(преф) − Close(обычка)** по синхронному бару (inner-join по времени обеих ног).
- Позиция **парная, дельта-нейтральная** (хедж из двух ног):
  - **LONG спреда** (ставка на РОСТ спреда) = `buy преф + sell обычка`
  - **SHORT спреда** (ставка на ПАДЕНИЕ спреда) = `sell преф + buy обычка`
- P&L зависит только от движения **спреда**, не от направления рынка (рыночный риск захеджирован).
- P&L по тикам: `(exit − entry) · dir · lots · (STEPPRICE / MINSTEP)`.

### Сигналы (Bollinger Bands на спреде)
- **Индикатор:** `SMA(period) ± sigma_mult · σ`, population-σ, no-repaint. Сигналы только на
  закрытии бара, когда BB прогрет (`is_ready`).
- **Вход — `Breakout`** (по умолчанию): пробой полосы наружу.
  - `SELL` (шорт спреда): `prev.spread < upper И cur.spread ≥ upper`
  - `BUY` (лонг спреда): `prev.spread > lower И cur.spread ≤ lower`
  - Альтернатива **`ReEntry`**: вход по ВОЗВРАТУ в канал (спред был снаружи и вернулся внутрь).
- **Гейт отклонения** (доп. фильтр входа, должен пройти вместе с пробоем):
  - `Sigma`: `|spread − SMA| ≥ deviation_sigma · σ`
  - `AbsOfMean`: `|spread − SMA| ≥ deviation_pct · |SMA|` (знаконезависим)
  - `LiteralPct`: `spread ≥ SMA·(1+pct)` (ломается при SMA<0, legacy)
- **Выход** (`exit_signal`): пересечение спредом средней SMA (возврат к среднему).
  - SHORT: `prev.spread > SMA И cur.spread ≤ SMA`
  - LONG: `prev.spread < SMA И cur.spread ≥ SMA`
  - SMA «живая» (дрейфует) или зафиксированная на входе (`freeze_sma_on_exit`).
- **Защитный стоп** (`stop_sigma`): если спред ушёл ПРОТИВ позиции дальше `stop_sigma · σ`
  от средней → закрытие по стопу. Защита от разрыва коинтеграции/тренда.

### FSM (конечный автомат)
`FLAT → ENTERING_{LONG,SHORT} → {LONG,SHORT}_SPREAD → EXITING → FLAT`, аварийный `HALTED`.
Максимум **1 открытая позиция**, повторный вход запрещён до возврата в FLAT.

## 3. Параметры (дефолты)

| Параметр | Значение | Назначение |
|---|---|---|
| `sma_period` | 200 | окно Bollinger Bands |
| `sigma_multiplier` | 2.0 | ширина полос (±σ) |
| `deviation_mode` | AbsOfMean | режим гейта входа |
| `deviation_pct` | 0.02 (2%) | порог для AbsOfMean/LiteralPct |
| `deviation_sigma` | 2.2 | порог для Sigma-режима |
| `entry_trigger` | Breakout | триггер входа |
| `stop_sigma` | 4.0 | защитный стоп |
| `take_profit_sigma` | 0.0 (**выкл**) | тейк-профит до пересечения SMA |
| `max_bars_in_trade` | 0 (**выкл**) | тайм-стоп |
| `volume_filter_mult` | 0.0 (**выкл**) | фильтр входа по объёму |
| `max_data_lag_min` | 0.0 (**выкл**) | фильтр свежести данных |
| `freeze_sma_on_exit` | false | фиксировать SMA на входе |
| `quantity_lots` | 1 | лотов на ногу (β=1) |
| MaxOpenPositions | 1 | одна позиция максимум |

**Per-pair оверрайды** (откалиброваны бэктестом):
- `sber`: дефолт (sma200 / σ2.0 / AbsOfMean 2%), 10m
- `sngr`: sma60 / σ1.5 / Sigma / deviation_sigma 1.5 / stop 4.0, 10m
- `rtkm`: sma100 / σ2.0 / Sigma / deviation_sigma 2.0, **60m**
- `tatn`: sma100 / σ2.0 / Sigma / deviation_sigma 2.0, **60m**

## 5. Известные проблемы и слабости

1. **Выход только по SMA** — позиция может висеть долго; SMA дрейфует за спредом. На тренде
   «возврат к средней» происходит из-за опускания самой средней, а не отскока спреда → убыток
   с reason=stop. Пример (sngr 24.06): вход лонг 23841, спред упал до 22856, стоп −585₽; без
   стопа выход был бы при 23299 = тоже убыток ~−545₽ (SMA уехала вниз вслед за спредом).

2. **Вход «на ноже»** — Breakout входит в разгар движения, не отличая шум от структурного
   сдвига. Нет фильтра тренда/разрыва коинтеграции.

3. **Тейк-профит, тайм-стоп, объёмный фильтр, фильтр свежести — ВЫКЛЮЧЕНЫ** по умолчанию.
   Потенциал не проверен на всех парах.

4. **Фиксированный размер позиции** (`quantity_lots` постоянный) — нет масштабирования под
   силу сигнала (|z-score|) или режим волатильности.

5. **Статичные пороги** входа и стопа — не адаптируются к режиму волатильности.

6. **β = 1 (равные лоты)** — предполагается единичный hedge ratio. Для пар с разным масштабом
   цен/лотов спред может быть несбалансирован (нет rolling-оценки коинтеграционного β).

7. **Неликвидность префов на мелких ТФ** — rtkm не торгует вовсе (~177 баров/90д → 0 сделок),
   tatn редкие сделки. Синхронизация ног рассыпается на 10m.

8. **Калибровка на одном периоде** — параметры подобраны одним бэктестом, без walk-forward /
   out-of-sample валидации на нескольких окнах.

## 6. Направления для улучшения (гипотезы для ИИ)

- **Адаптивные пороги** входа/стопа по режиму волатильности (σ-régime, ATR).
- **Динамический сайзинг** позиции по |z-score| на входе (больше отклонение → больше лот).
- **Включить и откалибровать** тейк-профит (частичная/полная фиксация до SMA) и тайм-стоп;
  проверить на всех парах бэктестом.
- **Фильтр структурного сдвига** перед входом: rolling ADF / Hurst exponent / тест на
  коинтеграцию — отсеивать входы перед разрывом (главная причина убытков).
- **Оценка hedge ratio β** через rolling-регрессию (Kalman / OLS) вместо фиксированных равных
  лотов — балансировать ноги динамически.
- **Walk-forward калибровка** per-pair вместо одного бэктеста; out-of-sample проверка.
- **Учёт стоимости исполнения** реалистичнее: `paper_book_halfspread_ticks` сейчас 1.0, реальный
  стакан префа шире — paper-P&L может быть оптимистичен.

## 7. Технические детали для воспроизведения

- Бэктест: `app/st4/backtest.py::run_backtest(df, cfg, spec_ord, spec_pref)` — форсит paper,
  считает equity-кривую с нереализованным P&L, честный maxDD.
- Эндпоинт: `GET /st4/backtest?days=N&stop_sigma=X&pair=PAIR` (через UI) или прямой вызов
  `run_backtest` (для смены `entry_trigger`, который через API не передаётся).
- Сигналы: `app/st4/strategy.py` (чистые функции). FSM/исполнение: `app/st4/engine.py`.
- Реестр пар и per-pair параметры: `app/st4/service.py::ST4_PAIRS`.
